# LifeOS · Control: Sleep (睡眠状态控制)

| 字段 | 内容描述 |
| :--- | :--- |
| **触发条件** | **节律驱动**：到达固定时间锚点（默认 22:00 / 23:00 / 07:00） |
| **适用范围** | 系统内所有主动运行的 Service（工作、学习、社交、运动、思考） |
| **判断规则** | **1. 预降级规则 (T > 22:00)**：强制释放“输入锁”，停止所有信息摄入；强制释放“责任锁”，不承诺新任务。<br>**2. 全局互斥规则 (T > 23:00)**：剥夺所有应用层 Service 执行权。系统进入“离线准备”，禁止自我评估与决策。<br>**3. 已知破坏事件 (KDE) 规则**：若发生强制性熬夜或外部干扰，标记系统为“降级模式 (Degraded Mode)”。<br>**4. 降级模式响应**：次日自动丢弃所有高消耗项（HIIT、深度脑力工作），不进行状态评分，不进行自我指责。 |
| **输出** | **[ Operating | Pre-Shutdown | Shutdown | Degraded ]** |

---

## 执行细节（Execution Details）

### 1. 预降级 (22:00 - 23:00)
- **Action**: 关闭社交媒体、关闭工作看板、关闭任何带有“进度条”的事物。
- **Focus**: 物理身体的撤防（调暗灯光、调节体温、轻微拉伸）。（加入9点半到娱乐）

### 2. 全局互斥 (23:00+)
- **Action**: 强制关闭“解决问题”的思考路径。
- **Mental Mode**: 若大脑持续跳出想法，将其视为“系统残留垃圾”，不予处理，直接丢弃。

### 3. KDE 处理流程
- **前提**：意识到今晚无法准时进入 Shutdown。
- **策略**：
    - 立即标记：`SYSTEM_STATE = DEGRADED`。
    - 隔离风险：不再试图通过“晚起”或“咖啡因”在次日进行等额补偿。
    - 路径：[次日] 运行最基础生存 Service -> 尽早回归标准节律锚点。
