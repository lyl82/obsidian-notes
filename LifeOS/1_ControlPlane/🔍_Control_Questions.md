# CONTROL 提问清单（决策前自检）

> 目标：在做【钱 / 健康 / 时间 / 关系 / 信息】相关的任何重要行动前，用一小组固定问题把冲动变成可见的结构。

---

## 0. 总体自检（任何重要决策都先问）

1. **现在的我，能量状态在 0–10 里大概是几分？**
2. **我现在是更接近「冷静」还是「兴奋 / 紧张 / 委屈」？**
3. **如果把决策推迟 24 小时，会有什么严重损失吗？具体是什么？**
4. **这件事是为了「逃避当前感受」还是为了「改善长期结构」？**
5. **这件事和 SERVICE 宪法冲突吗？（特别是：能量稳定、注意力稀缺、系统完整性）**

> 只要第 1–2 题显著偏向低能量/高情绪，或第 3 题无法给出清晰损失，默认进入「延迟」策略。

---

## 1. 钱相关决策（消费 / 投资 / 借贷）

1. **这笔支出/投资属于哪一类？**
   - 生存性 / 功能性 / 发展性 / 体验性 / 结构性？
2. **如果不做这笔支出，最坏会发生什么？是「不适」还是「真正的风险」？**
3. **如果把预算砍掉 50%，我还能接受什么样的方案？**
4. **这笔钱是否在侵占未来几个月的安全垫？（例如房租、基本生活费）**
5. **冲动来自哪里？是广告、他人晒收益，还是我长期思考后的决定？**

> 如果第 2 题的「最坏情况」只是情绪层面的 FOMO，而不是生存/结构性风险，优先延迟。

---

## 2. 健康相关决策（熬夜 / 运动 / 饮食）

1. **这件事会直接破坏我的睡眠质量吗？**
2. **如果今天熬夜/暴饮暴食，未来 72 小时内有哪些任务会被连带拖垮？**
3. **我现在是更需要「恢复」还是更需要「表现」？**
4. **有没有同类但更温和、可持续的替代方案？（例如：低强度锻炼替代 HIIT）**
5. **我是在奖励自己，还是在惩罚自己？**

> 如果答案里出现「我先透支一下，之后再补」「难得一次」，把它视为风险信号而不是理由。

---

## 3. 时间相关决策（答应任务 / 安排日程）

1. **这件事在本周的优先级是 S / A / B / C 中哪一档？为什么？**
2. **如果我答应了它，要牺牲掉哪一件已经在队列里的事情？**
3. **这件事是否必须由我来做？能否委托、延后或缩小范围？**
4. **如果把它拆成两个更小的版本，我会更容易开始吗？**
5. **我答应它，是因为真心想做，还是因为不好意思拒绝？**

> 只要第 2 题牺牲的是 S 级或 A 级长期目标，默认拒绝或重新谈条件。

---

## 4. 关系相关决策（冲动发言 / 重要对话 / 断联与复合）

1. **我现在最想表达的是「事实」还是「情绪」？**
2. **如果我把这条消息原封不动地发出去，一周后回看会不会后悔？**
3. **我到底想要什么结果？是被理解、被安慰、赢一场争论，还是切断某种模式？**
4. **这次互动会加深长期关系资本，还是只是一时的情绪宣泄？**
5. **有没有更低成本的尝试方式？（例如先写草稿、先打电话而不是长文、先问对方是否方便聊）**

> 若同时感到「很想马上说清楚」+「其实也有点怕对方反应」，这通常意味着需要冷却和草稿，而不是立刻行动。

---

## 5. 信息与成瘾相关决策（短视频 / 游戏 / 色情 / 刷信息流）

1. **我现在打开它，是因为无聊、疲惫，还是因为确实有具体目的？**
2. **过去 3 天里，我在这类行为上平均花了多少时间？**
3. **如果这一小时不用在屏幕娱乐上，我最想尝试的替代行为是什么？**
4. **每次结束这类行为后，我的典型感受是什么？（充实/空虚/后悔）**
5. **我是在主动选择，还是已经进入「自动模式」？**

> 若第 2 题的时间已经超出自己能接受的阈值，且第 4 题典型感受是后悔，默认进入限制模式（明天总时长 ≤ 30 分钟 + 提前列出替代行为）。

---

## 6. 结构性变更（搬家 / 辞职 / 远程旅行 / 重仓投资 / 关系重大调整）

1. **这件事会改变我未来 3 个月以上的生活结构吗？具体改变什么？**
2. **如果什么都不做，维持现状 3 个月，最坏的结果是什么？**
3. **如果失败了，我还能接受怎样的生活？有备选方案吗？**
4. **我已经独立思考过几次？是否在不同的日子、不同情绪下都写过？**
5. **这件事是我主动选择的「走向哪里」，还是被动逃离「现在的痛苦」？**

> 若第 4 题答案是「只想过一次/两次」，或主要动机是「受不了现在」，那就是信号：先做信息收集和小规模试探，而不是一次性 All-in。

---

## 7. 使用建议

- **最小使用法**：每次只选当前场景下最关键的 3 个问题写一句话回答，而不是全部回答。
- **记录位置**：将问题和答案简短记录在 DAO 日志或当日笔记中，作为之后复盘和 SERVICE 修改的依据。
- **低能量优先级**：当你很累时，只保留「0. 总体自检」中的前 3 个问题，其他一律简化为「可以晚点再决定吗？」。
- **不追求完美**：这些问题不是为了得到完美答案，而是为了把你从自动反应状态拉回到可见、可修改的结构中。
