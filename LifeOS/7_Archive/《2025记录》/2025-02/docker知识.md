### 1. 什么是 Docker？

Docker 就像一个超级厉害的 “打包工具”。你可以把应用程序（比如网页服务器、数据库软件）和它需要的所有环境（比如操作系统、配置文件、依赖文件等）打包成一个小盒子，这个小盒子就叫容器。就像你把一堆东西放进快递箱里，然后就可以很方便地搬运，不管搬到哪里，都能正常工作。

### 2. 为什么要用 Docker？

以前安装软件的时候，可能会遇到各种问题，比如不同电脑上的操作系统版本不一样，安装包装不上；或者装上了一个软件，却因为系统里已经有了其他东西，导致冲突。Docker 就能解决这些问题，它让软件和环境完全隔离，就像在不同的房间里装不同的东西，互不干扰。

### 3. Docker 的基本概念

- **镜像（Image）**：就像一个软件的模板，或者说是制作快递箱的设计图纸。比如你想在 Docker 里运行一个网页服务器（Nginx），就需要一个 Nginx 镜像。
    
- **容器（Container）**：用镜像启动后，运行的一个实例，就是容器。就像用快递箱模板做出来的一个箱子，里面装好了东西，可以运行了。
    

### 4. 怎么安装 Docker？

不同系统的安装方式不一样。

- **Windows 和 Mac**：去 Docker 官网 ([https://www.docker.com/](https://www.docker.com/)) 下载 Docker Desktop，然后按提示安装，一般就能用了。
    
- **Linux**：稍微复杂一点，需要用命令行安装，尤其是 Ubuntu 系统。不过你可以找一些简单的教程，比如在网上搜索 “Ubuntu 安装 Docker”，跟着一步一步操作，一般也不难。



### 5. 运行第一个容器

安装好 Docker 后，你可以试试运行一个简单的容器。

#### **拉取镜像**

在电脑打开命令行或者终端，输入：

bash复制

```bash
docker pull nginx
```

这个命令会从网上下载一个叫 Nginx 的镜像。Nginx 是个很常见的网页服务器软件，用来搭建网站。

#### **启动容器**

下载好镜像后，输入：

bash复制

```bash
docker run -d -p 80:80 --name myweb nginx
```

解释一下：

- `docker run`：表示启动一个容器。
    
- `-d`：让容器在后台运行，不会占用当前的终端窗口。
    
- `-p 80:80`：把电脑的 80 端口和容器的 80 端口 “连接” 起来，这样你通过浏览器访问 `http://localhost`，就能看到容器里运行的网页。
    
- `--name myweb`：给容器起个别名，叫 `myweb`，方便以后操作。
    
- `nginx`：告诉 Docker，用 Nginx 镜像来启动这个容器。
    

启动完成后，打开浏览器，访问 `http://localhost`，如果能看到一个 “Welcome to nginx!” 的界面，就说明你成功运行了第一个容器！

### 6. Docker 的简单操作

- **查看正在运行的容器**：
    
    bash复制
    
    ```bash
    docker ps
    ```
    
    这个命令会列出当前正在运行的容器。
    
- **停止容器**：
    
    bash复制
    
    ```bash
    docker stop myweb
    ```
    
    这里的 `myweb` 是容器的名字（上面我们用 `--name` 定义的）。停止后，容器就不会运行了。
    
- **删除容器**：
    
    bash复制
    
    ```bash
    docker rm myweb
    ```
    
    停止的容器可以删除，节省电脑的资源。
    

### 7. Dockerfile 是什么？

Dockerfile 是一个文件，里面写了一些指令，告诉 Docker 怎么制作一个镜像。你可以把它想象成一个 “食谱”，按照这个食谱，就能做出你想要的容器。比如，你想在容器里安装 Python，就可以在 Dockerfile 里写：

dockerfile复制

```dockerfile
FROM ubuntu:latest
RUN apt-get update && apt-get install -y python3
CMD ["python3", "app.py"]
```

这段代码的意思是：

- 用最新的 Ubuntu 系统作为基础。
    
- 在系统里安装 Python。
    
- 启动时运行 `app.py` 脚本。
    

### 8. 了解 Docker 网络和存储

- **网络**：容器之间或者容器与外面的电脑需要通信，就像房子之间有通道一样。Docker 默认会用 “桥接” 的方式，让容器能互相访问。
    
- **存储**：容器里的数据默认会随着容器停止而丢失，但你可以用 Docker 的数据卷功能，把数据保存在电脑的硬盘里，这样即使容器停了，数据也不会丢。
    

### 9. 总结

Docker 是一个很强大的工具，能帮你打包和运行软件，减少环境配置的烦恼。刚开始学的时候可能会觉得有点复杂，但慢慢熟悉命令和概念后，就会发现很方便！你可以先按照上面的步骤试一试运行容器，遇到问题可以在网上搜索，或者问我！
